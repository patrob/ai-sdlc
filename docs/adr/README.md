# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for ai-sdlc.
ADRs capture significant architectural choices, the context in which they were
made, and the rationale behind them.

## Index

| # | Title | Status | Date |
|---|-------|--------|------|
| [ADR-001](./001-esm-and-vitest-test-infrastructure.md) | ESM-first module system with Vitest and unit/integration split | Accepted | 2025-12 |
| [ADR-002](./002-provider-agnostic-agent-harness-via-pi.md) | Provider-agnostic agentic harness via Pi over direct Claude SDK | Accepted | 2026-02 |
| [ADR-003](./003-folder-per-story-storage.md) | Folder-per-story storage architecture | Accepted | 2026-01 |

## Format

Each ADR follows the lightweight format:
- **Status**: Proposed / Accepted / Deprecated / Superseded
- **Context**: The situation that required a decision
- **Decision**: What was decided
- **Consequences**: What results from this decision (positive and negative)
